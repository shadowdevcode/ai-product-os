# Codebase Context: smb-bundler
Last updated: 2026-03-19

## What This App Does

Enables B2B SaaS PMs to assemble a custom feature bundle from a 10-item catalogue and instantly receive a Gemini-generated value-based pricing proposal (INR range + 3 ROI bullet points) and a warm email pitch tailored for Indian SMB buyers. The PM selects 1‚Äì10 features, clicks "Generate", and can copy the email pitch directly into Gmail or WhatsApp ‚Äî target flow: landing to copied pitch in under 2 minutes. No auth. Analytics are dual-tracked via PostHog events and a `bundle_sessions` table in Neon DB (the `pitch_copied` boolean is the primary hypothesis validation signal).

## Architecture Overview

- **Frontend**: Next.js 16 App Router, TypeScript strict, Tailwind CSS 4, Framer Motion. Single page (`app/page.tsx`) holds all state ‚Äî `selectedIds`, `sessionId`, `proposal`, `loading`, `error`. All components are Client Components.
- **Backend**: Two API routes ‚Äî `POST /api/generate-proposal` (validate ‚Üí Gemini ‚Üí Neon insert ‚Üí respond) and `PATCH /api/bundle-sessions/[id]/copied` (mark pitch copied).
- **Database**: Neon DB (serverless PostgreSQL). `@neondatabase/serverless` HTTP driver (no connection pool needed). Single table: `bundle_sessions`. Schema in `schema.sql` ‚Äî must be applied manually in Neon SQL Editor before first deploy.
- **AI Integration**: Gemini 2.5 Flash via `@google/genai` with structured JSON schema output. AbortController at 9s ‚Äî returns JSON 504 on timeout. In-memory rate limiter (5 req/60s per IP) protects Gemini cost surface.
- **Analytics**: PostHog server-side (`posthog-node`) for `bundle_generated`, `proposal_generation_failed`, `proposal_generation_timeout`, `proposal_generation_rate_limited`. PostHog client-side (`posthog-js`) for `pitch_copied` and `landing_page_viewed`.

## Data Model

**`bundle_sessions`** ‚Äî one row per generation event.

| Column | Type | Notes |
|---|---|---|
| `id` | UUID PK | Pre-generated by caller via `crypto.randomUUID()` before DB insert |
| `selected_features` | TEXT[] | Array of feature label strings (e.g. `["Automated Invoicing", "GST-ready Reports"]`) |
| `price_range_label` | TEXT | Gemini-generated label (e.g. "‚Çπ2,499 ‚Äì ‚Çπ3,499/month") |
| `generated_price_inr` | INTEGER | Gemini numeric price, clamped to 999‚Äì9999 server-side |
| `estimated_monthly_savings_inr` | INTEGER | Gemini-generated savings figure |
| `roi_points` | TEXT[] | Exactly 3 ROI bullet strings |
| `email_pitch` | TEXT | Full ~150-word email pitch |
| `pitch_copied` | BOOLEAN | Updated to true via PATCH ‚Äî primary hypothesis validation signal |
| `created_at` | TIMESTAMPTZ | Auto-set to NOW() |

No `user_id` ‚Äî anonymous sessions, no auth, no RLS.

## Things NOT to Change Without Reading First

- **`crypto.randomUUID()` is called BEFORE `insertBundleSession()`** in `route.ts`. This is intentional ‚Äî the ID must be stable even if Neon fails. Do not move the UUID generation to after the DB call.
- **Rate limiter is in-memory** (`rateLimitMap` in `route.ts`). It resets on cold start and is per-Vercel-instance. This is an accepted MVP limitation ‚Äî do not replace with Redis without reading the postmortem.
- **`flushAt: 1`** on the PostHog server client is intentional for MVP (ensures events fire on serverless). It adds ~100ms latency per event; accepted tradeoff.
- **DB failure is non-blocking** ‚Äî `insertBundleSession` is wrapped in try/catch and proposal response is returned regardless. Do not change this to a blocking flow.
- **`FEATURE_LABELS_WHITELIST`** (server) and `FEATURES` (client) must stay in sync. If you add a feature, update both in `lib/features.ts`.
- **Gemini price clamping**: `monthly_price_inr` is clamped to `[999, 9999]` server-side but `price_range_label` is NOT recalculated (known accepted issue RR3 from peer review). Fix this if Gemini frequently returns out-of-range prices.

## Known Limitations

- `price_range_label` may diverge from clamped `monthly_price_inr` if Gemini returns an out-of-range price (RR3, accepted MVP tradeoff)
- In-memory rate limiter is per-Vercel-instance ‚Äî bots routing across instances can exceed 5 req/60s (accepted MVP limitation)
- No client-side fetch timeout ‚Äî infinite spinner on network stall (QA6, low severity)
- `roi_points` has no `minItems` guard ‚Äî empty array from Gemini produces a degraded proposal, no error (QA edge case, low severity)
- `bundle_sessions` table grows unbounded ‚Äî no TTL or archival strategy (SC2 from peer review)
- `[First Name]` in email pitch is a literal placeholder ‚Äî amber warning note is shown above the pitch but PM must remember to replace it
